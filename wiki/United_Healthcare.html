Connectivity

Using a Representational State Transfer (REST) connection, API data is returned in a JavaScript Object Notation (JSON) format. There are two connectivity options for API:

  1. Link API: Connect to us directly through a secure connection for all available API types.
  2. API Extended X12: API data is returned as an extension to EDI transaction responses. Currently available for claim status only, but eligibility and benefits will be available soon.

View the guides for each API connection at UHCprovider.com/apiguides

Getting Started

Link API:

  • Send an email to [[email protected]?subject=Link API consultation request|[email protected]] to request a consultation. Include your name, title, organization name, email address and phone number. We will contact you to review options, compatibility and connection terms.
  • Additional Steps for Health Care Providers:
    • Register in Link for an Optum ID, if you haven't already. Refer to the New User Registration GuideOpens in a new window for instructions.
    • Register all Tax IDs connected to your organization by requesting multi-TIN accessOpens in a new window.
    • The Password Owner of your organization needs to log in using their Optum ID to ensure it's a current, active account.

API Extended X12:

  • To establish connectivity for API Extended X12, a direct connection to UnitedHealthcare is required.
  • A direct connection is not required if you are receiving API data from your clearinghouse. Contact your clearinghouse to inquire about the UnitedHealthcare data they have available through their API services.
  • If you're interested in an API Extended X12 connection, send an email to [[email protected]?subject=API Extended X12 Connection request|[email protected]] to request a consultation. Include your name, title, company name, email address and phone number. We will contact you to discuss API options, compatibility and connection terms.
